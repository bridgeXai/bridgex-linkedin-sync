# BridgeX LinkedIn Sync · 团队安装与使用指南

> **版本**：2026-05-26  
> **适用对象**：BridgeX 各团队 LinkedIn 触达同事  
> **代码仓库**：https://github.com/BridgeX-ai/bridgex-linkedin-sync  
> **数据去向**：所有团队写入**同一个**飞书多维表格（表 C 明细 + 表 B 日汇总）

---

## 一、这个插件做什么？

Chrome 浏览器插件，用于**半自动**记录你在 LinkedIn 上的触达行为：

- 你在 LinkedIn 上**真人**浏览、发连接、发私信
- 插件在你点击按钮时，把目标人信息和动作**同步到飞书**
- **不会**自动发消息、不会静默爬取整站（降低封号风险）

写入目标：

| 飞书表 | 用途 | 链接 |
|--------|------|------|
| **表 C** Reach out CRM | 每次触达一条明细 | [打开表 C](https://dcnzdjjl3pwl.feishu.cn/base/JaCgbEeGRagC7KsYJtNc2XLPnMN?table=tbl7pWyLK5f15UyO) |
| **表 B** 每日操作日志 | 按 LinkedIn 账户汇总当天计数 | [打开表 B](https://dcnzdjjl3pwl.feishu.cn/base/JaCgbEeGRagC7KsYJtNc2XLPnMN?table=tblvmhRYACuRixCB) |

## 二、安装前准备

| 项目 | 要求 |
|------|------|
| 浏览器 | **Google Chrome**（或 Chromium 内核，如 AdsPower 内置浏览器） |
| 操作系统 | Windows / macOS 均可 |
| 网络 | 能访问 LinkedIn、飞书、OpenRouter |
| 飞书权限 | 你本人能**查看/编辑**上述多维表格（如无法打开链接，联系管理员开通） |
| **`lib/secrets.js`** | **向管理员私聊索取**（GitHub 仓库里不含凭证，勿自行乱填） |
| Node.js | **可选**，仅收工跑表 B 纠偏脚本时需要（18+） |

## 三、获取插件（二选一，约 2 分钟）

### 方式 A：从 GitHub 下载（推荐，方便以后更新）

**第 1 步**：打开仓库  
https://github.com/BridgeX-ai/bridgex-linkedin-sync

**第 2 步**：点击绿色 **Code** 按钮 → **Download ZIP**

**第 3 步**：解压到本地固定路径，例如：

- Windows：`D:\Tools\bridgex-linkedin-sync\`
- macOS：`~/Tools/bridgex-linkedin-sync/`

**第 4 步**：确认解压后能看到这些文件：

```
D:\Tools\bridgex-linkedin-sync\
├── manifest.json          ← 必须有
├── popup.html
├── options.html
├── lib\
│   └── secrets.example.js ← 模板（还不能用）
└── 安装指南.md             ← 就是本文档
```

> **注意**：加载扩展时选择的是**包含 `manifest.json` 的这一层文件夹**，不要多套一层。

**第 5 步**：向管理员索取 **`secrets.js`**，放到 `lib\` 目录下：

```
lib\secrets.js   ← 管理员私发，放这里
```

放好后 `lib\` 目录应同时有 `secrets.js` 和 `secrets.example.js`。

---

### 方式 B：用 Git 克隆（会 Git 的同事）

打开 PowerShell 或终端，复制粘贴运行：

```powershell
git clone https://github.com/BridgeX-ai/bridgex-linkedin-sync.git D:\Tools\bridgex-linkedin-sync
cd D:\Tools\bridgex-linkedin-sync
```

然后同样向管理员索取 `lib\secrets.js` 放进目录。

**以后更新**只需：

```powershell
cd D:\Tools\bridgex-linkedin-sync
git pull
```

再到 `chrome://extensions/` 点 **重新加载**（见 §十）。

## 四、安装步骤（约 5 分钟）

### 步骤 1：确认文件齐全

打开你的插件文件夹，检查：

- [ ] 有 `manifest.json`
- [ ] 有 `popup.html`
- [ ] 有 `lib\secrets.js`（管理员给的，**不是** `secrets.example.js`）

> **安全提示**：`secrets.js` 内含团队 API 凭证，**仅限 BridgeX 内部使用**，勿转发给外部人员。

---

### 步骤 2：在 Chrome 加载插件

1. 打开 Chrome，地址栏输入并回车：

   ```
   chrome://extensions/
   ```

2. 打开右上角 **「开发者模式」** 开关

3. 点击左上角 **「加载已解压的扩展程序」**

4. 选择你的插件文件夹（含 `manifest.json` 的那一层）

5. 列表中出现 **「BridgeX LinkedIn Sync」** → 安装成功 ✅

6. （建议）点击扩展图标旁的 📌，**固定到工具栏**

> 若使用 **AdsPower**：在对应浏览器配置文件的扩展管理页同样操作。

**装完应长这样：**

```
chrome://extensions/
┌─────────────────────────────────────┐
│  BridgeX LinkedIn Sync              │
│  从当前 LinkedIn 标签页读取...       │
│  [详细信息]  [重新加载]  [移除]      │
└─────────────────────────────────────┘
```

---

### 步骤 3：填写个人身份（必做，每人不同）

1. 右键浏览器工具栏上的 BridgeX 图标 → **「选项」**  
   （或：`chrome://extensions/` → BridgeX → **详细信息** → **扩展程序选项**）

2. 填写以下两项：

| 设置项 | 填什么 | 示例 |
|--------|--------|------|
| **操作者（真人同事）** | 你的真实姓名 | 李靖、朱婧雯 |
| **LinkedIn 账户** | 当前浏览器登录的 LinkedIn 人设代号 | `BRGX001`、`MVP-01` |

3. 确认 **时间字段 = `sent_at`**（不要填 `reply_at`）

4. **`sent_at` 写入格式**：
   - 飞书列是「日期」或「日期时间」→ 选 **日期时间戳**（默认）
   - 飞书列是「文本」→ 选 **日期文本**

5. 点击 **保存**

> **重要**：「操作者」和「LinkedIn 账户」是两层身份，**不能混填**。  
> 例：李靖用 Jason Park 账户操作 → 操作者=李靖，账户=BRGX001。

---

### 步骤 4：验证安装（必做，确认能写飞书）

1. 用 AdsPower / Chrome 打开任意 LinkedIn **个人主页**（不是搜索页、不是消息列表）

2. 点击工具栏上的 **BridgeX 插件图标**

3. Popup 应自动显示：

   - 姓名
   - 职位
   - 公司
   - 地区
   - LinkedIn URL

4. 点 **「已发申请」** 测试同步

5. 打开 [表 C](https://dcnzdjjl3pwl.feishu.cn/base/JaCgbEeGRagC7KsYJtNc2XLPnMN?table=tbl7pWyLK5f15UyO)，看是否多了一行，`operator_name` 是否是你的姓名

**全部 OK → 安装完成，可以开始日常使用了。**

若报 **403 / Forbidden** → 见 §九，联系管理员开飞书文档权限。

## 五、共享飞书表格说明

**所有团队共用同一张多维表格**，通过以下字段区分数据来源：

| 字段 | 作用 | 谁填 |
|------|------|------|
| `operator_name` | 真人操作者 | 插件选项页 → 操作者 |
| `account_id` | LinkedIn 人设账户 | 插件选项页 → LinkedIn 账户 |
| `sent_at` | 触达日期（精确到日） | 插件自动写入当天 |
| `status` | 动作类型 | 插件按钮决定 |

**表 B 日汇总**按 `account_id` + 当天日期聚合，因此：

- 换 LinkedIn 账户操作时，**必须先改选项页的 LinkedIn 账户**
- 同一账户多人轮流操作可以，但 `operator_name` 会记录每次是谁点的

## 六、日常使用流程

### 流程 A：发连接申请

1. 打开目标人 LinkedIn 主页
2. （推荐）点 **「📸 履历整页截屏」** 采集工作经历 / 教育背景
3. 核对姓名、公司、职位、**地区**
4. 在 LinkedIn 上**手动**发送连接申请
5. 点插件 **「已发申请」** → 写入表 C，表 B `connect_sent + 1`

### 流程 B：发私信（完整字段）

1. 主页 → **「📸 履历整页截屏」**（页面过长可**多次**截屏，自动合并）
2. 复制已发送的私信全文 → 粘贴到 popup **「发出的消息」**
3. 点 **「✨ AI 分类」** → 预览 Pain / Offer / CTA
4. 点 **「已发消息」**（**消息框为空会被拦截，不会同步**）
5. 到表 C 核对 `message_sent`、`career_background`、`pain_angle` 等

### 流程 C：对方接受连接

1. 打开**同一人**的 LinkedIn 主页
2. 点 **「连接已接受」** → 表 C 该行 status 改为 Connected，表 B `connect_accepted + 1`

### 流程 D：收到回复

1. 打开对方主页
2. 点 **「已回复」**

### 右键快捷操作

在 LinkedIn 页面**划选文字**后右键：

| 菜单 | 用途 |
|------|------|
| BridgeX 同步此信息作为履历备注 | 填入履历素材框 |
| BridgeX 保存为发出的消息 | 填入「发出的消息」框 |

> 日常操作可对照 **[使用说明-快速参考.md](./使用说明-快速参考.md)** 打印或钉在桌面旁。

## 七、五个动作按钮对照

| 插件按钮 | 表 C status | 表 B 计数 |
|----------|-------------|-----------|
| 已发申请 | Sent | connect_sent + 1 |
| 已发消息 | Messaged | messages_sent + 1 |
| 已回复 | Replied | replies_received + 1 |
| 连接已接受 | Connected（更新已有行） | connect_accepted + 1 |
| 其他 | Sent | connect_sent + 1 |

## 八、每次操作前检查（养成习惯）

- [ ] 当前 LinkedIn 账户与选项页 **LinkedIn 账户** 一致
- [ ] 在目标人 **个人主页**（不是搜索页 / 消息页）
- [ ] 已在 LinkedIn **手动**完成发连接 / 发消息
- [ ] 发私信时：消息已粘贴到 popup（点「已发消息」前）

## 九、常见问题

| 现象 | 处理方法 |
|------|----------|
| **403 / 91403 Forbidden** | 飞书 Base 未授权应用。联系管理员：Base 右上角「…」→「更多」→「添加文档应用」→ 授予**可编辑** |
| **插件加载失败 / 找不到 manifest** | 选错文件夹了。必须选含 `manifest.json` 的那一层 |
| **没有 lib/secrets.js** | 向管理员私聊索取，放到 `lib\` 目录，然后重新加载扩展 |
| **表 B 没更新** | 检查选项页 LinkedIn 账户是否填对；或联系管理员跑表 B 纠偏脚本 |
| **地区为空** | 手动在 popup 补填 |
| **Background 有，Pain/Offer 空** | 正常——Pain/Offer 来自**发出的消息**，不是履历。先粘贴消息 → AI 分类 → 再同步 |
| **点「已发消息」没反应** | 消息框为空会被拦截，必须先粘贴消息正文 |
| **同步后字段比预览少** | Popup 底部 → **查看解析/同步记录**，看错误详情 |
| **换 LinkedIn 账户忘了改设置** | 立即到选项页修改 LinkedIn 账户，后续操作才会记到正确账户 |
| **更新代码后插件没变化** | `chrome://extensions/` → BridgeX → 点 **重新加载**（不会自动更新） |

## 十、版本更新

### Git 用户

```powershell
cd D:\Tools\bridgex-linkedin-sync
git pull
```

### zip 用户

1. 从 GitHub 重新 **Download ZIP**，或向管理员要新版 zip
2. **先备份** 旧目录里的 `lib\secrets.js`
3. 解压覆盖（或解压到新路径）
4. 把备份的 `secrets.js` 拷回 `lib\`

### 所有人

1. 打开 `chrome://extensions/` → BridgeX → 点 **「重新加载」**
2. 检查选项页「操作者」「LinkedIn 账户」是否还在（一般会自动保留）
3. **不要删除** `lib\secrets.js`
